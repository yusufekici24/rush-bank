# PR: Unity 6 Portrait Prototype Scene and Visual Layout Pass

## Summary

This PR moves RushBank from script-level prototype work into an actual Unity Editor playable scene flow. It fixes Unity `6000.5.4f1` compatibility issues, adds portrait mobile project configuration, generates the prototype scenes, and updates the first `Game` scene composition for a vertical 9:16 mobile screen.

The current prototype now opens in Unity, runs in Play Mode, shows UI, moves the player character, and has no red Console errors after the latest validation pass.

## What Changed

### Unity 6.5 compatibility fixes

- Replaced deprecated `Object.GetInstanceID()` usage in `ChubbyCharacterDresser` with a stable local seed helper.
- Replaced obsolete built-in `Arial.ttf` calls with `LegacyRuntime.ttf` across runtime UI and prototype setup code.
- Added missing `RushBank.Core` namespace imports for gameplay systems that reference `TimeManager`.
- Fixed `VIPCustomer` edit-mode material mutation by using `sharedMaterial` with an explicit cloned material.
- Added a local helper script, `tools/OpenRushBankUnity.ps1`, to open the project with a safer Unity launch environment on this Windows machine.

### Portrait mobile setup

- Added `RushBank > Configure Android Portrait Settings`.
- The tool configures:
  - app name
  - package identifier
  - portrait orientation
  - disabled landscape orientations
  - Android target switch when Android Build Support is available
- It now gracefully warns when Android Build Support is not installed instead of throwing a red Console error.

### URP, TMP, and generated Unity assets

- Added Universal Render Pipeline dependency to `Packages/manifest.json`.
- Included `Packages/packages-lock.json`.
- Imported TMP essentials generated by Unity.
- Included Unity-generated `.meta` files and ProjectSettings needed for the editor/project state.
- Included generated prototype scenes and ScriptableObject data:
  - `Boot`
  - `Login`
  - `MainMenu`
  - `Game`
  - request/customer/transaction prototype assets
  - transaction prefabs

### Prototype scene generation

- Updated `RushBankPrototypeSetup` so `RushBank > Setup Prototype Scenes` works on Unity `6000.5.4f1`.
- Generated the full scene set under `Assets/Scenes`.
- Updated runtime UI canvas and HUD sizing for portrait mobile.
- Kept the `Game` scene suitable for quick Play Mode validation.

### Portrait 9:16 visual composition

- Reworked the `Game` scene layout for a vertical mobile screen:
  - player and main teller counter sit near the bottom of the screen
  - counter is close to the `Call Customer` button area
  - middle screen space is reserved for customer flow and waiting activity
  - top screen area shows the branch entrance
  - security guard home position moved next to the entrance
  - left side has two specialist desk areas
  - right side has two specialist desk areas
  - center area has vertical waiting benches and a small table
- Extended the floor toward the bottom of the portrait frame so UI buttons sit over game space instead of empty background.
- Adjusted orthographic camera size/position for broader portrait visibility.

### Visual tone

- Reduced the earlier overly bright color palette.
- Shifted the prototype toward a warmer, calmer "Tasra branch" look.
- Increased mobile HUD/button sizing for touch readability.
- Improved side desk and waiting lounge readability for future referral mechanics.

## Why This Changed

The design/story side of RushBank is mostly defined, so the project needed to move into Unity scene staging. Before adding more visual polish, the first goal was to make the project:

- open correctly in Unity `6000.5.4f1`
- compile cleanly
- generate scenes
- run in Play Mode
- display a portrait mobile layout
- provide a readable first branch composition

The fixes in this PR unblock future work on visual polish, real scene dressing, customer flow tuning, and MVP gameplay validation.

## Player/Developer Impact

- Developers can now run `RushBank > Setup Prototype Scenes` and open `Assets/Scenes/Game.unity`.
- The prototype can be viewed in `9:16` Game View.
- The first playable scene gives a clearer mobile composition:
  - bottom: player and teller counter
  - middle: customer waiting/activity area
  - sides: referral desks
